# Справочник Боевой Математики

Этот документ собирает боевые правила проекта в одном месте. Значения вдохновлены референсной таблицей, но адаптированы под наш прототип, чтобы контент оставался понятным и удобно настраивался в Unity.

## База Персонажа

- Здоровье стандартного героя: `120`.
- Сила атаки: умножает урон навыка. По умолчанию `1.00`.
- Шанс критического удара: вероятность умножить итоговый урон навыка.
- Критический урон: множитель критического удара. По умолчанию `2.00`.
- Частота применения навыков: делит кулдаун. Значение `2.00` означает, что навыки применяются в два раза чаще.
- Множитель области: умножает итоговый радиус области поражения.
- Двойная атака / мультикаст: бросает дополнительные применения навыка с каскадным множителем `0.4`.
- Защита: переводится в снижение урона через формулу брони.
- Радиус подбора: управляет расстоянием, с которого герой подбирает кристаллы опыта.
- Тотем жизни: один заряд предотвращает смерть и восстанавливает 50% здоровья.

## Формула Урона

Урон навыка считается так:

```text
УронСРангом = БазовыйУрон + (Ранг - 1) * ПриростУронаЗаРанг
СыройУрон = УронСРангом * СилаАтаки * МножительУронаВладельца + ПлоскийБонусУронаНавыка
ИтоговыйУрон = СыройУрон * КритическийМножитель, если прошёл бросок крита
```

Кулдаун считается так:

```text
КулдаунСРангом = БазовыйКулдаун - (Ранг - 1) * СнижениеКулдаунаЗаРанг
ИтоговыйКулдаун = max(0.15, КулдаунСРангом * МножительКулдауна / ЧастотаНавыков)
```

В таблице видно, что отображаемое значение кулдауна упирается примерно в `0.2s`, но реальный нижний предел применения ближе к `0.15s`. Поэтому в проекте жёсткий минимум кулдауна равен `0.15s`.

## Формула Брони

Для врагов `100` брони считается нейтральным значением.

```text
если Броня >= 100:
    МножительУрона = 2 / ((Броня / 100) + 1)
иначе:
    МножительУрона = (200 - Броня) / 100
```

Защита игрока считается как броня выше нейтральной точки:

```text
ЭффективнаяБроня = 100 + Защита
ВходящийУрон = СыройУрон * МножительУронаОтБрони(ЭффективнаяБроня)
```

## Реализация Конструктора Атак

Теперь атака в проекте собирается из трёх переиспользуемых блоков:

- Targeting: решает, куда или в кого должна быть направлена атака.
- Delivery: решает, каким способом удар доставляется до цели.
- Impact: решает, что удар делает с целью.

Реализованные блоки доставки удара:

| Тип Delivery-ассета | Для чего подходит | С чем комбинировать |
| --- | --- | --- |
| Projectile | одиночное копьё, стрела, болт, ядовитый плевок | Nearest Enemy, Random Enemies |
| Projectile Spread | много ножей, веер копий, дробовой залп ядом | Nearest Enemy |
| Area Pulse | топот, падение метеора, удар молнии, взрыв по земле | Self Position, Random Enemies |
| Frontal Cone | удар секирой, рассечение, укол копьём, фронтальная волна | Nearest Enemy |
| Chain | цепная молния, прыгающий теневой удар | Nearest Enemy, Random Enemies |
| Lasting Area | ядовитая лужа, поле бури, огненная зона, святая земля | Random Enemies, Self Position |

Примеры сборки:

- Бросить много ножей: `Nearest Enemy Targeting` + `Projectile Spread Delivery` + `Damage Impact`.
- Удар секирой: `Nearest Enemy Targeting` + `Frontal Cone Delivery` + `Damage Impact`.
- Бросить валун: `Nearest Enemy Targeting` + `Projectile Delivery` с высоким уроном и меньшей скоростью.
- Ядовитая лужа: `Random Enemies Targeting` + `Lasting Area Delivery` + `Damage Impact` + `Status Impact(Poison)`.
- Грозовая молния: `Random Enemies Targeting` + `Area Pulse Delivery` или `Chain Delivery` + `Damage Impact`.
- Укол копьём: `Nearest Enemy Targeting` + `Frontal Cone Delivery` с узким углом.

## Правила Статусов

Текущие ассеты проекта хранят `potency` статуса как урон за тик или процентный модификатор. Мы сохраняем это ради совместимости.

- Poison: периодический урон, складывается в стаки.
- Burn: периодический урон, складывается в стаки.
- Bleed: физический периодический урон, складывается в стаки.
- Doom: заготовка под отложенный/тикающий урон. Позже должен взрываться или наносить дополнительный урон при завершении статуса или смерти цели.
- Slow: снижает скорость движения на `potency * stacks`.
- Weakness: снижает исходящий урон и контактный урон.
- Shattered: напрямую снижает броню.
- Dazed: увеличивает стат критического шанса на поражённой цели. Позже это лучше заменить на отдельную механику “шанс крита по этой цели”, когда добавим теги входящего урона.
- Brittle: сейчас мягко снижает броню. Позже должен добавлять плоский дополнительный урон за каждое попадание.
- Fragility: снижает исходящий урон. Позже должен стать механикой “повышенный входящий урон”.

## Рост Силы Врагов

Здоровье врага:

```text
ИтоговоеЗдоровье = БазовоеЗдоровье * МножительТаймлайна * КриваяВремени * МножительКатегории
КриваяВремени = 1 + НормализованноеВремяРана^1.35 * 1.45
```

Урон врага:

```text
ИтоговыйУрон = БазовыйУрон * МножительТаймлайна * КриваяВремени * МножительКатегории
КриваяВремени = 1 + НормализованноеВремяРана^1.15 * 0.75
```

Множители категорий:

| Категория | Здоровье | Урон |
| --- | ---: | ---: |
| Normal | 1.00 | 1.00 |
| Empowered | 1.35 | 1.15 |
| Elite | 3.50 | 1.35 |
| MiniBoss | 12.00 | 1.75 |
| Boss | 35.00 | 2.50 |

Логика спавна должна заставлять врагов входить с окружности вокруг игрока, а не появляться в случайных точках внутри карты. Сложность должна расти через бюджет спавна, смесь категорий врагов, аффиксы и множители таймлайна, а не только через сырое количество врагов.

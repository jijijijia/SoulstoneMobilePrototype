# Общая Математика Игры

Этот документ описывает все базовые формулы и балансовые принципы для 30-минутного забега.

---

## Философия Баланса

- Первые 5 минут: игрок учится, враги слабые, прирост силы ощутимый.
- 5–15 минут: сборка билда, давление растёт, нужно уже думать.
- 15–25 минут: высокая плотность, элитки, позиционирование важно.
- 25–30 минут: интенсивная финальная фаза, боссы, нельзя расслабляться.
- Сложность растёт через плотность, состав волн и комбинирование паттернов, а не через ваншот урон.
- HP врагов растёт быстрее урона. Скорость растёт умеренно.

### Итоговый Стат

```text
finalStat = (baseValue + additiveModifiers) * (1 + sumMultiplierModifiers)
```

## Формула Урона Навыка

```text
rankedDamage = baseDamage + (rank - 1) * damagePerRank

totalAttackPower = max(0.05, attackPower * ownerDamageMultiplier)

rawDamage = max(1, rankedDamage * totalAttackPower + flatDamageBonus)

if random(0,1) <= critChance:
    finalDamage = round(rawDamage * max(1.0, critMultiplier))
else:
    finalDamage = round(rawDamage)
```

---

## Формула Кулдауна

```text
rankedCooldown = max(0.15, baseCooldown - (rank - 1) * cooldownReduction)

finalCooldown = max(0.15, rankedCooldown / max(0.1, skillFrequency))
```

Жёсткий минимум кулдауна: **0.15s**.

---

## Формула Области Поражения

```text
finalRadius = max(0.05, (baseRadius + flatAreaBonus) * max(0.1, areaMultiplier))
```

---

## Формула Брони

Броня 100 = нейтральная точка (урон без изменений).

```text
если armor >= 100:
    damageMultiplier = 2 / ((armor / 100) + 1)
иначе:
    damageMultiplier = (200 - armor) / 100
```

Примеры:
| Броня | % Урона |
| ---: | ---: |
| 50 | 150% |
| 100 | 100% |
| 150 | 80% |
| 200 | 67% |
| 300 | 50% |

Защита игрока переводится в броню:
```text
effectiveArmor = 100 + defense
incomingDamage = rawDamage * damageMultiplier(effectiveArmor)
```

## Повторное Применение (DoubleAttack / Multicast)

```text
stepChance = doubleAttackChance * 0.4^executionIndex
maxExecutions = 8
```

Пример: doubleAttackChance = 0.3 (30%):
- Попытка 1: 30%
- Попытка 2: 30% * 0.4 = 12%
- Попытка 3: 30% * 0.16 = 4.8%

Среднее ожидаемое количество применений ≈ 1.43 при 30%.

## Масштабирование Врагов По Времени

Нормализованное время: `t = currentTime / runDuration`, от 0 до 1.

### HP Врагов

finalHP = baseHP * timelineMultiplier * timeCurveHP * categoryMultiplier

## Формула Эффективного DPS Навыка

```text
skillDPS = (baseDamage * attackPower + flatBonus) * critMult * (1/finalCooldown)

critMult = 1 + critChance * (critMultiplier - 1)
```

Пример для среднего навыка (urон 55, КД 2.0, AttackPower 1.0, Crit 5%, CritMult 2.0):
```text
critMult = 1 + 0.05 * 1.0 = 1.05
DPS = (55 * 1.0) * 1.05 / 2.0 = 28.9
```

Средний суммарный DPS игрока при 6 навыках ≈ 120–200 единиц/сек к концу забега.

## Время Убийства (TTK)

```text
TTK = enemyFinalHP / playerTotalDPS
```

Целевые значения:

| Цель | Начало (0 мин) | Середина (15 мин) | Конец (30 мин) |
| --- | --- | --- | --- |
| Обычный враг | 0.3–0.7 сек | 0.5–1.0 сек | 0.8–1.5 сек |
| Элитный враг | 2–4 сек | 4–8 сек | 8–15 сек |
| Мини-босс | 8–15 сек | 15–25 сек | 25–40 сек |
| Босс | — | 30–60 сек | 45–90 сек |

## Прокачка Навыков По Уровням

Каждый навык имеет 5 уровней. Примерная схема прироста:

```text
Уровень 1: базовые значения
Уровень 2: +урон или -кулдаун
Уровень 3: +радиус или +количество целей/снарядов
Уровень 4: +дополнительный эффект (статус, pierce, ricochet)
Уровень 5: +сильное улучшение (смена паттерна или x1.5 к параметру)
```

Суммарный прирост к 5-му уровню: x2.0–x3.0 эффективности.

## Контрольные Точки Баланса

Признаки слишком лёгкой игры:
- Обычный враг умирает с 1 удара к 10-й минуте.
- Игрок не получает урон 3+ минуты подряд.
- TTK обычного врага меньше 0.2 сек в любое время.

Признаки слишком сложной игры:
- Обычный враг убивает игрока за 3 удара в первые 5 минут.
- Враги с AoE перекрывают весь экран в первые 10 минут.
- TTK элитного врага превышает 30 секунд при хорошем билде.
- Игрок не успевает уклоняться от большинства атак.

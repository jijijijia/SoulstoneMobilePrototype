# Гайд По Созданию Контента

Этот документ отвечает на практический вопрос: как руками создавать персонажей, оружие, навыки, улучшения, карты и врагов в текущей data-driven архитектуре проекта.

## Общий Принцип

Контент создаётся через ассеты в `Assets/Data`, а сцены должны оставаться максимально общими. Идея такая: сцена запускает ран, а конкретный герой, оружие, карта, враги и навыки приходят из данных.

Главная цепочка персонажа:

```text
CharacterData
-> WeaponData
-> SkillData
-> AttackDefinition
-> Targeting + Delivery + Impact
```

Главная цепочка карты:

```text
MapData
-> Enemy Pool
-> SpawnTimelineData
-> SpawnSystem
```

Главная цепочка повышения уровня:

```text
ProgressionSystem
-> ProgressionChoiceGenerator
-> либо 3 активных навыка
-> либо 3 улучшения характеристик
```

Важно: в одном окне повышения уровня навыки и улучшения не смешиваются. Игра случайно выбирает тип награды целиком: либо показывает 3 навыка, либо 3 улучшения. Если один из типов недоступен, система использует доступный тип.

## Карта Проекта

```text
Assets/Data
-> Characters      данные героев
-> Weapons         данные оружия
-> Skills          данные активных навыков
-> Upgrades        данные улучшений
-> Enemies         данные врагов
-> Maps            данные карт
-> AttackModules   модули атак: targeting, delivery, impact, definitions

Assets/Scripts/DataDriven
-> Attacks         модульная система атак
-> Runtime         рантайм-данные и выборы прогрессии
-> Skills          запуск навыков
-> Systems         герой, прогрессия, спавн, враги, статусы
-> UI              HUD, level up, main menu

Assets/Scenes
-> MainMenu        главное меню
-> main            игровая сцена
```

## Персонаж

Персонаж отвечает за тело героя: живучесть, скорость, сбор опыта и защитные механики. Он не должен хранить уникальную механику атак. Атаки живут в навыках, а доступ к навыкам задаётся через оружие.

Путь:

```text
Assets/Data/Characters
```

Чтобы создать персонажа:

1. Открой `Assets/Data/Characters`.
2. Дублируй похожий `CharacterData`.
3. Переименуй ассет, например `Hero_Viking`.
4. Заполни `Character Id`, `Display Name`, описание и модель.
5. Настрой базовые характеристики.
6. Укажи `Starting Weapon`.
7. Добавь героя в `Assets/Resources/CharacterRoster.asset`, чтобы он появился в меню персонажей.

Актуальные характеристики персонажа:

| Характеристика | Что делает |
| --- | --- |
| `MaxHealth` | Максимальное здоровье героя |
| `MoveSpeed` | Скорость передвижения |
| `PickupRadius` | Радиус подбора опыта |
| `Defense` | Снижает входящий урон |
| `ParryChance` | Шанс полностью заблокировать атаку |
| `ExperienceMultiplier` | Множитель получаемого опыта |
| `DashCharges` | Количество рывков до перезарядки |
| `HealthRegenPercent` | Регенерация здоровья в процентах |
| `LifeTotemCount` | Количество воскрешений после смертельного урона |

Боевые характеристики навыков, такие как урон, крит, область, кулдаун и повторное применение, должны настраиваться в `SkillData` или через улучшения.

## Оружие

Оружие определяет стартовый активный навык и пул навыков, которые могут выпадать этому оружию во время забега.

Путь:

```text
Assets/Data/Weapons
```

Чтобы создать оружие:

1. Открой `Assets/Data/Weapons`.
2. Дублируй похожий `WeaponData`.
3. Переименуй ассет, например `Weapon_VikingAxe`.
4. Заполни:
   - `Weapon Id` - уникальный id.
   - `Display Name` - имя оружия.
   - `Icon` - иконка.
   - `Starting Skill` - стартовый активный навык.
   - `Unique Skill Pool` - список активных навыков, которые могут выпадать этому оружию.
5. Назначь это оружие персонажу в `CharacterData`.

Если навык не добавлен в `Unique Skill Pool`, он не будет выпадать при повышении уровня для этого оружия.

## Навык

Навык отвечает за конкретную атаку: урон, кулдаун, элемент, тип удара, визуал и ссылку на модульную механику атаки.

Путь:

```text
Assets/Data/Skills
```

Актуальные поля навыка:

| Поле | Что делает |
| --- | --- |
| `Skill Id` | Уникальный id навыка |
| `Display Name` | Название навыка |
| `Description` | Описание для UI |
| `Icon` | Иконка |
| `Attack Type` | Общий тип удара |
| `Element` | Стихия или элемент |
| `Runtime Definition` | Какая рантайм-логика запускает навык |
| `Visual Prefab` | Визуальный префаб снаряда, зоны или эффекта |
| `Base Damage` | Базовый урон |
| `Flat Damage Bonus` | Плоский бонус к урону |
| `Crit Chance` | Шанс критического удара |
| `Crit Multiplier` | Множитель критического урона |
| `Area Bonus` | Плоский бонус к области поражения |
| `Cooldown` | Базовый кулдаун |
| `Cooldown Coefficient` | Базовый коэффициент кулдауна |
| `Cooldown Divider` | Делитель кулдауна |
| `Area Multiplier` | Множитель области поражения |
| `Repeat Chance` | Шанс повторного применения навыка |

`Runtime Definition` - это ссылка на логику, которая умеет запускать навык. Для новых модульных атак обычно используется `ModularAttackSkillRuntimeDefinition`, а сама механика собирается в `AttackDefinition`.

## Конструктор Атак

Сейчас атака собирается из трёх частей:

```text
AttackDefinition
-> Targeting
-> Delivery
-> Impact
```

`Targeting` отвечает за выбор точки или цели.  
`Delivery` отвечает за то, как атака появляется и движется.  
`Impact` отвечает за последствия: урон, статусы и другие эффекты.

### Targeting

Путь:

```text
Assets/Data/AttackModules/Targeting
```

| Targeting | Что делает |
| --- | --- |
| `Nearest Enemy` | Выбирает ближайшего врага |
| `Random Enemy` | Выбирает случайного врага |
| `Self Position` | Использует позицию игрока |

### Impact

Путь:

```text
Assets/Data/AttackModules/Impact
```

| Impact | Что делает |
| --- | --- |
| `Damage` | Наносит урон |
| `Status` | Накладывает статус: яд, горение, замедление и так далее |

Одна атака может иметь несколько impact-модулей. Например, удар может одновременно наносить урон и накладывать яд.

### Delivery

Путь:

```text
Assets/Data/AttackModules/Delivery
```

Delivery - это главный “вид атаки”. Если хочется сделать новый визуально и механически другой навык, чаще всего меняется именно delivery.

| Delivery | Что делает | Примеры |
| --- | --- | --- |
| `Projectile` | Выпускает один снаряд | стрела, копьё, яд |
| `Projectile Spread` | Выпускает веер или залп снарядов | метание клинков, залп копий |
| `Area Pulse` | Мгновенно бьёт по области | удар грома, взрыв |
| `Delayed Area` | Показывает точку удара и бьёт после задержки | метеорит, армагедон |
| `Lasting Area` | Создаёт длительную зону урона | ядовитая лужа, огненная зона |
| `Chain` | Бьёт несколько целей последовательно | серия молний |
| `Frontal Cone` | Бьёт сектором перед героем | удар мечом, фронтальный взмах |
| `Line Slash` | Бьёт вытянутой зоной | двойной секущий удар |
| `Circular Melee` | Мгновенный круговой удар вокруг героя | удар топором вокруг себя |
| `Circular Sweep` | Имитация вращающегося удара по настраиваемой дуге | 360-градусный взмах секирой, 120-градусный удар мечом |
| `Beam` | Луч или линия урона | луч света, тёмный разрез |
| `Orbital` | Создаёт вращающийся вокруг игрока объект | клинки вокруг героя, молоты |
| `Summon` | Призывает союзника с HP, жизнями, лимитом количества и простым ИИ | скелет, волк, голем |

## Продвинутые Снаряды

`Projectile` и `Projectile Spread` поддерживают дополнительные параметры:

| Параметр | Что делает |
| --- | --- |
| `Pierce Count` | Сколько врагов снаряд может пробить перед исчезновением |
| `Ricochet Count` | Сколько раз снаряд может перескочить на новую цель |
| `Ricochet Search Radius` | Радиус поиска следующей цели для рикошета |
| `Fork Generations` | Сколько раз снаряд может разделиться после попадания |
| `Fork Projectile Count` | Сколько дочерних снарядов создаётся при разделении |
| `Fork Spread Angle` | Общий угол разлёта дочерних снарядов |

Как это использовать:

```text
Обычная стрела:
Pierce Count = 0
Ricochet Count = 0

Пробивающее копьё:
Pierce Count = 4
Ricochet Count = 0

Рикошетящий кинжал:
Pierce Count = 0
Ricochet Count = 3
Ricochet Search Radius = 8

Разветвляющийся фаербол:
Fork Generations = 1
Fork Projectile Count = 2
Fork Spread Angle = 35
```

Для мобильной оптимизации не делай каждый снаряд тяжёлым физическим объектом. Снаряды должны быть простыми триггерами и использовать `PoolManager`, а не постоянные `Instantiate/Destroy`.

## Типы Атак По Игровой Логике

Ниже список типов, которые уже можно собирать в нашем проекте через текущие delivery-модули.

| Механика | Как собрать |
| --- | --- |
| Одиночный снаряд | `Projectile` |
| Веер снарядов | `Projectile Spread` |
| Пробивающий снаряд | `Projectile` + `Pierce Count > 0` |
| Рикошетящий снаряд | `Projectile` + `Ricochet Count > 0` |
| Разветвляющийся снаряд | `Projectile` + `Fork Generations > 0` |
| Удар вокруг героя | `Circular Melee` |
| Круговой взмах по дуге | `Circular Sweep` |
| Фронтальный удар сектором | `Frontal Cone` |
| Длинный секущий удар | `Line Slash` |
| Мгновенный взрыв по области | `Area Pulse` |
| Удар с задержкой | `Delayed Area` |
| Длительная зона | `Lasting Area` |
| Цепная молния | `Chain` |
| Луч | `Beam` |
| Орбитальные объекты | `Orbital` |
| Призыв союзника | `Summon` |

`Summon` сейчас создаёт боевого союзника. У него есть здоровье, количество жизней, скорость, радиус атаки, интервал атаки, длительность существования и лимит активных существ. Враги умеют отвлекаться на ближайшего призванного союзника в радиусе агро и могут его убить. Позже эту систему можно расширить отдельными типами поведения питомцев, анимациями и разными атаками призванных существ.

## Как Создать Новый Навык

Лучший путь - не писать новый скрипт, а собрать навык из существующих модулей.

1. Создай или дублируй `Targeting`.
2. Создай или дублируй `Impact`.
3. Создай или дублируй `Delivery`.
4. Создай `AttackDefinition` и укажи в нём targeting, delivery и impacts.
5. Создай `SkillData`.
6. В `SkillData` укажи `Runtime Definition` для модульной атаки.
7. Привяжи `AttackDefinition`.
8. Настрой урон, кулдаун, крит, область и повторное применение.
9. Добавь навык в `Unique Skill Pool` нужного оружия.

Пример: “Армагедон”.

```text
SkillData: Skill_Armageddon
AttackDefinition: Attack_Armageddon
Targeting: Random Enemy
Delivery: Delayed Area
Impact: Damage
Visual Prefab: большой камень или маркер падения
```

Пример: “Метание клинков”.

```text
SkillData: Skill_BladeStorm
AttackDefinition: Attack_BladeStorm
Targeting: Nearest Enemy
Delivery: Projectile Spread
Impact: Damage
Ricochet Count: 1-3, если нужны прыгающие клинки
```

Пример: “Взмах секирой”.

```text
SkillData: Skill_AxeSweep
AttackDefinition: Attack_AxeSweep
Targeting: Self Position
Delivery: Circular Sweep
Impact: Damage
Sweep Angle: 360
Arc Width: 70
```

Пример: “Удар мечом по сектору”.

```text
SkillData: Skill_SwordArc
AttackDefinition: Attack_SwordArc
Targeting: Self Position
Delivery: Circular Sweep
Impact: Damage
Sweep Angle: 120
Arc Width: 55
```

## Улучшения

Улучшения должны быть универсальной болванкой-конструктором.

Путь:

```text
Assets/Data/Upgrades
```

У улучшения должны быть:

| Поле | Что делает |
| --- | --- |
| `Upgrade Id` | Уникальный id |
| `Display Name` | Название |
| `Description` | Описание |
| `Rarity` | Редкость и частота выпадения |
| `Target` | Что улучшаем: героя, все навыки, конкретный навык или группу навыков |
| `Stat` | Какую характеристику меняем |
| `Value` | Значение изменения |

Логика таргета:

| Target | Что означает |
| --- | --- |
| `Character` | Улучшает героя: здоровье, скорость, защиту, опыт, рывки и так далее |
| `All Skills` | Улучшает все активные навыки |
| `Specific Skill` | Улучшает только один указанный навык |
| `Skill Tag / Element` | Улучшает навыки с нужным тегом или элементом |

Примеры:

```text
+15% урона всем навыкам:
Target = All Skills
Stat = Damage
Value = 0.15

+25% урона только Армагедону:
Target = Specific Skill
Skill = Skill_Armageddon
Stat = Damage
Value = 0.25

+1 рывок герою:
Target = Character
Stat = DashCharges
Value = 1
```

## Повышение Уровня

Текущая логика:

1. Игрок набирает опыт.
2. `ProgressionSystem` фиксирует новый уровень.
3. Игра ставится на паузу.
4. `ProgressionChoiceGenerator` случайно выбирает тип окна:
   - 3 активных навыка;
   - или 3 улучшения.
5. Игрок выбирает одну награду.
6. Игра продолжается.

Важное правило: в одном окне не должно быть одновременно навыков и улучшений. Если выпали навыки, все 3 карточки - навыки. Если выпали улучшения, все 3 карточки - улучшения.

Активные навыки выпадают уже как полноценные навыки, а не как “уровень 1 навыка”. Механику прокачки конкретного навыка будем делать отдельно.

## Карты

Путь:

```text
Assets/Data/Maps
```

Карта должна задавать:

| Поле | Что делает |
| --- | --- |
| `Map Id` | Уникальный id карты |
| `Display Name` | Название |
| `Scene Name` | Какую сцену загрузить |
| `Description` | Описание |
| `Biome` | Биом |
| `Difficulty` | Сложность |
| `Enemy Pool` | Какие враги могут появляться на карте |
| `Spawn Timeline` | Как растёт сложность и какие волны появляются |

Если хочешь уникальных врагов для карты, добавляй их в `Enemy Pool` конкретной `MapData`. Тогда другая карта сможет иметь другой набор врагов без копирования сцены.

## Враги

Путь:

```text
Assets/Data/Enemies
```

Враг должен задавать:

| Поле | Что делает |
| --- | --- |
| `Enemy Id` | Уникальный id |
| `Display Name` | Название |
| `Prefab` | Префаб врага |
| `Health` | Базовое здоровье |
| `Damage` | Базовый урон |
| `Move Speed` | Скорость |
| `Experience Reward` | Сколько опыта даёт |
| `Attack Behavior` | Как атакует: ближний бой, дальняя атака, яд и так далее |

Для врагов с дальними атаками важно, чтобы они не подходили вплотную, а останавливались на дистанции и использовали свой навык с задержкой.

## Производительность

Правила, которые важно соблюдать для survivors-like игры:

1. Не использовать `FindObjectsOfType` в рантайм-циклах атак.
2. Использовать реестр активных врагов, например `EnemyRegistry`.
3. Использовать `sqrMagnitude` вместо `Vector3.Distance`, если нужна только проверка ближайшей цели.
4. Использовать `PoolManager` для снарядов, эффектов, врагов и опыта.
5. Не создавать и не удалять сотни объектов через `Instantiate/Destroy` каждый кадр.
6. Не делать каждый визуальный эффект тяжёлым физическим объектом.
7. Для толп врагов избегать лишней физики и сложных коллайдеров.

## Быстрый Чеклист Создания Контента

Новый герой:

```text
CharacterData
-> Starting Weapon
-> CharacterRoster
```

Новое оружие:

```text
WeaponData
-> Starting Skill
-> Unique Skill Pool
-> CharacterData
```

Новый навык:

```text
Targeting
-> Impact
-> Delivery
-> AttackDefinition
-> SkillData
-> WeaponData.Unique Skill Pool
```

Новая карта:

```text
MapData
-> Enemy Pool
-> Spawn Timeline
-> Main Menu map list
```

Новый враг:

```text
EnemyData
-> Enemy Prefab
-> Enemy Pool нужной карты
```

## Как Понять, Где Что Менять

Если хочешь изменить цифры героя - ищи `CharacterData`.  
Если хочешь изменить список доступных навыков - ищи `WeaponData`.  
Если хочешь изменить урон, кулдаун или крит навыка - ищи `SkillData`.  
Если хочешь изменить механику атаки - ищи `AttackDefinition` и его `Delivery`.  
Если хочешь изменить эффект попадания - ищи `Impact`.  
Если хочешь изменить выпадение наград - ищи `ProgressionSystem` и `ProgressionChoiceGenerator`.  
Если хочешь изменить врагов карты - ищи `MapData` и `Enemy Pool`.  
Если хочешь изменить рост сложности - ищи `SpawnTimelineData`.
